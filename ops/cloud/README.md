# Internet Demo: Render + LiveKit Cloud

This setup gives you a public app URL and managed voice without self-hosting LiveKit.

## 1) Create LiveKit Cloud project

In LiveKit Cloud create a project, then copy:
- Project URL (`wss://<project>.livekit.cloud`)
- API key
- API secret

## 2) Deploy with Render Blueprint

This repo already has `/Users/mohamed.darwish/Documents/New project/render.yaml`.

On Render:
- New + -> Blueprint
- Select this repo
- Render creates:
  - `discordclone-app` web service
  - `discordclone-db` Postgres

Set these required env vars in the web service:

```env
LIVEKIT_URL=wss://<project>.livekit.cloud
LIVEKIT_API_KEY=<livekit-api-key>
LIVEKIT_API_SECRET=<livekit-api-secret>
```

Everything else is preconfigured by `render.yaml`:
- `DATABASE_URL` from managed Postgres
- `JWT_SECRET` autogenerated
- `JWT_EXPIRES_MIN=4320`
- `LIVEKIT_TOKEN_TTL_MIN=240`

## 3) Verify

After deploy:
- `https://<your-render-app-domain>/health` -> `{"status":"ok"}`
- `https://<your-render-app-domain>/health/ready` -> `database=true, voice=true`

Share:
- `https://<your-render-app-domain>/`

## Notes

- `DATABASE_URL` values in `postgres://`/`postgresql://` form are normalized automatically.
- CORS auto-allows local dev and Render hostname by default.
